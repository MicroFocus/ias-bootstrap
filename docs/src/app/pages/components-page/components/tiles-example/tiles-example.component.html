Tiles

Tiles are a navigation component that allow a user to access pages, applications, and other data within a product. At minimum a tile should possess a recognizable avatar or icon image. Tiles may also be given titles and descriptions depending on the size of the tile.

Tiles serve as group-able navigational components with recognizable elements.

The default style of an element using the .ias-tile class. Tiles allow for a familiar icon, descriptive title, and a short description of an element in their styling by default. Tiles are a responsive element and will adapt their layout to the size of their containing element.

Tiles can be implemented using anchor tags to create clickable link. The "Micro Focus" examples below make use of the link implementations of a tile.

The smallest version of the tile is implemented by added the .ias-tile-small class to an .ias-tile element. These elements make use of an icon and a descriptive title attribute to describes the tile's purpose.

Medium tiles are a horizontal variant implemented by adding the .ias-tile-medium class to an .ias-tile element. These tiles have a familiar icon as well as a short descriptive title.

Tiles can be laid out in a wrapping grid using the CSS Grid layout. See the definition for the "app-grid" class for an example. Use the slider bar to the right of the grid below to see the grid adjust to fill the available space.
